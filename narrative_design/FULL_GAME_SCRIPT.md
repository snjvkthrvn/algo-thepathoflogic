## HOW TO READ THIS SCRIPT

- **[STAGE]** = Visual/environmental directions
- **[AUDIO]** = Music and sound cues
- **[SYSTEM TEXT]** = Text that appears as if printed on the world itself
- **[NARRATION]** = Text overlay, like a system log the player reads
- **PLAYER CHOICE** = Branching dialogue options
- **[TRANSITION]** = Scene change descriptions
- **[PUZZLE]** = Gameplay segment with pre/post dialogue
- *Italics* = Stage directions and internal actions
- `Code blocks` = System data displays

### [PUZZLE P0-1: FOLLOW THE PATH]

**Mechanic:** The rune tiles glow in a sequence (like Simon Says). The player must step on them in the correct order. Each round adds one more step to the sequence.

**Visual States:**
- **Inactive:** Gray (#4a5568), 60% opacity, gentle bob
- **Glowing (in sequence):** Bright cyan (#06b6d4), particle burst, rising chime whose pitch varies with tile index
- **Stepped correctly:** Golden outline (#fbbf24), white center, satisfying "click-hum" sound
- **Stepped incorrectly:** Red flash, dissonant buzz + glass crack, tile resets

**Rounds:**
- Round 1: 3-step sequence
- Round 2: 5-step sequence
- Round 3: 7-step sequence

**DSA Concept:** Sequential processing — arrays as ordered steps, following instructions in exact order.

**[AUDIO during puzzle]:** Each tile produces a different musical note when stepped on, creating a melody when the sequence is completed correctly. Correct full sequence triggers bright ascending arpeggio.

### [PUZZLE P0-2: FLOW CONSOLES]

**Mechanic:** The player must pick up crystal shards and place them in the correct console terminals. Each shard has a shape+stripe combination that matches exactly one console. Incorrect placement produces a dissonant buzz and the shard bounces back.

**Shard Types:**
- Shard A: Triangle + double stripes → Red Console
- Shard B: Diamond + single stripe → Blue Console
- Shard C: Circle + triple stripes → Green Console

**Visual Feedback:**
- Picking up a shard: It hovers near the player, glowing
- Approaching correct console: Soft bloom glow, harmonic resonance increases
- Correct placement: Satisfying "crystal snap" — the shard locks in, energy flows to central core
- Incorrect placement: Shard bounces back, dissonant buzz, brief red flash

**Central Core States:**
- 0 shards placed: Gray, inactive
- 1 shard placed: Dim glow on one connection point
- 2 shards placed: Two connection points lit, energy starting to flow between them
- 3 shards placed: Full activation — brilliant cyan light, energy lines connecting all points

**DSA Concept:** Key-value mapping — each shard (key) maps to exactly one console (value). Direct access vs. searching.

# ═══════════════════════════════════════════
# APPENDIX: DIALOGUE STYLE GUIDE
# ═══════════════════════════════════════════

## Professor Node
- Warm, encouraging, never condescending
- Uses computing terms naturally ("Your state seems good," "Let me cache that thought")
- Excited about knowledge — crystal spins faster when enthusiastic
- Underneath the warmth: carries a weight he doesn't share with the player

## The Pattern
- Always structured data format
- Logic, never emotion
- Asks questions frequently — genuinely tries to understand what it can't categorize
- Never threatens — states conclusions ("You must be collected" is logical, not hostile)

## The Echo
- Player's voice but archaic, fragmented
- Sentences occasionally repeat—repeat or stutter
- Enormous weariness — the exhaustion of running too long
- Final conversation feels like a parent speaking to a child who will outlive them

## Village Elder
- Deep, slow, patient
- Nature metaphors mixed with computing metaphors
- "An array is just a row of boxes" — makes the complex simple

## NPCs in Loops
- Mechanical repetition — same words, same inflection, same timing
- When freed: wonder and confusion, like waking from a dream
- Computing metaphors used naturally ("I need to process this," "My memory is fragmented")

## The Architect (Written Only)
- Clear, precise, slightly formal
- Genuine warmth beneath technical language
- Acknowledges uncertainty — doesn't pretend to have all answers
- Trusts the reader
