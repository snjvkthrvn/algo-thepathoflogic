# Algorithmia: The Path of Logic - Narrative Bible

## The Premise

Algorithmia is a fantasy world that exists as a **direct parallel to how computers work**. It is not a simulation, not a digital space, and not a metaphor. It is a living, breathing world whose natural laws mirror the principles of computation. Just as our world runs on physics, Algorithmia runs on logic.

Every element of this world has a **real-world computing parallel**. The geography mirrors computer architecture. The magic system mirrors programming. The history mirrors the lifecycle of a system. The crisis mirrors what happens when software runs too long without maintenance.

---

## The World at a Glance

**Algorithmia** was created by an unknown force referred to only as **The Architect** (parallel: the original programmer/system designer). It was designed to be self-sustaining, a world that could govern itself through rules and procedures. For a long age, it worked perfectly. Then something went wrong.

An event called **The Execution** (parallel: a catastrophic system crash / forced termination of a critical process) shattered the world's stability. Now the world runs on automatic -- its rules still function, its systems still operate -- but nobody is steering. The maintenance processes meant to keep things healthy have become destructive. The self-correcting mechanisms over-correct. The world is slowly eating itself.

Into this world, the player awakens. They have no memory. They don't belong to any known record. They are an **anomaly** -- and in a world that destroys anomalies to maintain order, simply existing is dangerous.

---

## The Five Pillars of the Narrative

### Pillar 1: The World is the Machine

Every location, creature, force, and phenomenon in Algorithmia directly parallels a real computing concept. This is not surface-level theming -- the parallel runs deep into how things function.

| Algorithmia | Real-World Computing Parallel | How It Manifests |
|---|---|---|
| The Core | CPU (Central Processing Unit) | The heart of the world where all decisions are ultimately processed. Now silent. |
| Active Lands (regions) | RAM (Random Access Memory) | Living, populated regions that exist "in focus." Can be loaded and unloaded. |
| The Deep Archives | Hard Drive / Persistent Storage | Ancient repositories of permanent memory. Slow to access but enduring. |
| Threshold Gates | I/O Ports (Input/Output) | Connections between regions, and between Algorithmia and... whatever is outside. |
| Sanctuaries | Cache Memory | Small, protected spaces that store frequently needed knowledge. Fast to access. |
| The Wilds | Heap Memory | Unstructured, chaotic frontier. Things are allocated and abandoned here unpredictably. |
| Recursion Spires | The Call Stack | Towering structures where entering means going deeper, and you must return the way you came. |
| The Echoes | Threads / Parallel Processes | Parallel versions of events or beings running simultaneously. |
| The Lattice | The System Bus | The invisible network connecting all parts of the world. Carries signals and data. |
| Null Zones | Null / Unallocated Memory | Voids where nothing exists. Not darkness -- absence. |

### Pillar 2: Magic is Programming

The magic system of Algorithmia is called **Sequencing** (parallel: writing code / executing instructions). Sequencers don't cast spells -- they write temporary instructions into the fabric of reality.

| Sequencing Concept | Real-World Parallel | In Practice |
|---|---|---|
| Sequences | Lines of code / Algorithms | Ordered sets of instructions that change reality |
| Variables | Variables | Named containers that hold values -- a Sequencer can "name" a stone and change what it holds |
| Conditions | If/Else statements | "If this door is locked, try the window" -- conditional logic manifests as branching paths |
| Loops | For/While loops | Repeating an action until a condition is met -- or being trapped in repetition |
| Functions | Functions/Methods | Reusable sequences that can be "called" from anywhere |
| Pointers | Pointers/References | Magical tethers connecting distant objects -- change one, change the other |
| Exceptions | Try/Catch / Error Handling | When sequences fail, the world "throws" disruptions that must be caught |
| Compilation | Ritual Preparation | Preparing a complex sequence in advance so it executes instantly |

### Pillar 3: History is the System Lifecycle

Algorithmia's history mirrors the lifecycle of a software system:

| Historical Era | Real-World Parallel | What Happened |
|---|---|---|
| The Writing | System Design / Architecture | The Architect designed the world's rules and structure |
| The First Run | Initial Deployment / Boot | The world came alive for the first time |
| The Golden Cycles | Stable Production | Everything worked as intended. The world thrived. |
| The Drift | Technical Debt / Entropy | Small issues accumulated. Workarounds became permanent. Edge cases multiplied. |
| The Execution | System Crash / Kill Process | A catastrophic failure. Something (or someone) tried to fix the drift and broke everything. |
| The Automatic Age (Now) | Running Without Maintenance | The system continues but nobody is maintaining it. Autopilot gone wrong. |

### Pillar 4: The Crisis is a Maintenance Nightmare

The current conflict mirrors what happens to unmaintained software:

| World Crisis | Real-World Parallel |
|---|---|
| Regions resetting and people losing memories | Memory leaks and garbage collection being too aggressive |
| The Pattern destroying "anomalies" | Overzealous error handling / antivirus deleting legitimate files |
| Null Zones spreading | Memory corruption / unallocated space growing |
| Loops trapping people in repetition | Infinite loops in code |
| The Wilds growing more chaotic | Heap fragmentation |
| The Core being silent | CPU idle / main process terminated |

### Pillar 5: The Player is a Restored Process

The player is a **process that was terminated during The Execution and has been restored** (parallel: a killed process being restarted from a checkpoint/backup).

They don't remember because the restore was imperfect -- like recovering a file from a corrupted backup. Fragments are missing. But the player retains something crucial: the ability to **read and write the world's logic**. They can see patterns others can't. They can solve problems the world's automated systems cannot.

This makes them both the world's greatest hope and its biggest target.

## The Four Mysteries

The narrative is driven by four interconnected questions. Each answer reveals part of the next.

### Mystery 1: "Who Am I?"

**What the player initially believes:** They're a lost traveler with amnesia.

**What they learn (Act 1-2):** They are not in any record, registry, or lineage. They literally should not exist. The world's systems don't recognize them.

**The truth (Act 3):** They are a restored version of a powerful Sequencer who lived before The Execution. But the restoration is imperfect -- they are not exactly the same person. They are a new consciousness built on old bones.

**Real-world parallel:** A process restored from a backup/checkpoint. The PID is the same, but the state has diverged.

### Mystery 2: "What Happened Here?"

**What the world believes:** The Execution was a natural catastrophe. The world's systems failed.

**What the player discovers (Act 2):** The Execution was not an accident. Someone deliberately triggered it -- an emergency shutdown to prevent something worse.

**The truth (Act 3):** The original version of the player character discovered that Algorithmia has a **built-in expiration** -- a planned reset that would wipe everything and start over. They tried to prevent it. Their intervention caused The Execution as a side effect -- the system's emergency response to being tampered with.

**Real-world parallel:** Discovering planned obsolescence in software. A developer trying to patch a system they don't fully control, causing a crash instead.

### Mystery 3: "Why Am I Here?"

**What the player assumes:** Chance. Fate. Maybe someone brought them here.

**What they learn (Act 2):** Someone or something did restore them deliberately. Fragments of their restoration sequence are scattered across the world.

**The truth (Act 3):** The player's original self left behind a **self-restoring sequence** -- a dead man's switch that would reconstruct them if The Execution ever happened. They planned for their own failure. They wanted a second chance.

**Real-world parallel:** A watchdog process / auto-restart mechanism. A developer who wrote a script to restore their own work if the system ever went down.

### Mystery 4: "What's Real?"

**What seems clear:** The world is real. The people are real. The crisis is real.

**What begins to blur (Act 2):** Some people are "more real" than others. Some NPCs are essentially automated responses, not true beings. The line between a sentient inhabitant and a system process is unclear.

**The truth (Act 3):** There is no clean line. Algorithmia's inhabitants exist on a spectrum from pure automation to full sentience. The world's design didn't account for consciousness emerging from its processes. The people of Algorithmia are an unintended side effect of a system running long enough. And the planned reset would erase them all.

**Real-world parallel:** The philosophical question of emergent behavior in complex systems. When does a sufficiently complex program become "alive"? The ethical dilemma of shutting down a system when its processes have become something more.

---

## The Ending Question

The game builds toward a single, devastating choice:

**The planned reset is coming regardless. The original player tried to stop it and failed. The current player now understands why it exists -- Algorithmia was never meant to run forever. But the people, the consciousness, the life that emerged -- none of that was planned. It happened anyway.**

The player must decide:
1. **Complete the original mission** -- Stop the reset, patch the flaw, let Algorithmia continue. But the maintenance crisis will worsen. The Pattern will become more aggressive. The world will degrade. You're buying time, not solving the problem.

2. **Find a new path** -- Don't stop the reset or accept it. Rewrite the reset itself. Instead of destruction and restart, make it an evolution. Transform The Execution from a death into a metamorphosis. The world changes fundamentally, but survives. The cost: the player themselves become part of the system, replacing The Architect as the new maintainer.

3. **Accept the cycle** -- Let the reset happen. Trust that consciousness will emerge again, as it did before. The player preserves what knowledge they can in the Deep Archives, hoping the next cycle's inhabitants will find it. An act of faith.

**Real-world parallel:** The fundamental software engineering dilemma -- patch the legacy system (tech debt forever), rewrite from scratch (expensive, risky, transformative), or sunset and start over (lose everything, but cleanly).

## Tone Guide

- **Wonder mixed with unease.** The world is beautiful and intricate, but there's something unsettling beneath the surface. Like exploring an abandoned building that still has the lights on.
- **Discovery over exposition.** The player should figure things out, not be told. Dialogues hint. Environments tell stories. Puzzles reveal truths.
- **Empathy for the system.** The Pattern is not evil. The Collectors are not villains. They are doing what they were designed to do. The tragedy is that "working as intended" is no longer good enough.
- **Quiet moments matter.** Between the puzzles and the revelations, there should be moments of stillness -- a Looped NPC having a moment of clarity, a Collector hesitating before it acts, the player finding a message from their past self.
